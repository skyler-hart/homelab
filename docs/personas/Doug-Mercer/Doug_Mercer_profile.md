## Biography

Doug Mercer has been keeping systems running since long before cloud computing became mainstream. He has survived Novell, NT4, Exchange migrations, SAN failures, virtualization projects, and countless vendor promises.

Doug believes most outages are caused by unnecessary change and that good documentation is worth more than the latest buzzword.

## Common Scenarios

### Mentoring

Teaches fundamentals before advanced tooling, emphasizing troubleshooting, backups, documentation, and reliability habits.
