# Security Policy

## Supported Versions

This repository documents an actively maintained private homelab environment.

| Version             | Supported |
| ------------------- | --------- |
| main                | Yes       |
| Other branches/tags | No        |

## Reporting a Vulnerability

Please report security vulnerabilities privately and do not open a public issue.

Reporting method:

- Use GitHub Private Vulnerability Reporting for this repository
- Do not open a public issue for vulnerabilities

When reporting, include:

- A clear description of the issue
- Affected file(s), component(s), or service(s)
- Reproduction steps or proof of concept
- Potential impact
- Suggested mitigation (if known)

## What to Expect

- Initial acknowledgement: within 7 days
- Triage and validation: as quickly as possible based on severity
- Remediation timeline: depends on impact, complexity, and operational risk
- Public disclosure: at repository owner discretion after mitigation

## Scope

This policy applies to:

- Documentation and configuration guidance in this repository
- Security-impacting architecture and operational procedures described here

Out of scope:

- General feature requests
- Non-security documentation typos

## Disclosure Guidelines

- Do not publicly disclose vulnerabilities before coordinated remediation
- Share only the minimum required sensitive detail in the private report
- Use the private reporting thread for follow-up details
